# Session File Format

Me Write Code stores sessions as JSONL. Each line is one JSON object with a `type` field. Entries form a tree through `id` / `parentId`, which enables `/fork`, `/tree`, and in-place branch navigation.

## Location

Default session files live under:

```text
~/.mewrite/agent/sessions/
```

Set `sessionDir` in settings or pass `--session-dir` to use another location.

## Versions

| Version | Meaning |
| --- | --- |
| 1 | Legacy linear sequence; auto-migrated on load |
| 2 | Tree structure with `id` / `parentId` |
| 3 | `hookMessage` role renamed to `custom` |

Existing sessions are migrated to the current version when loaded.

## Header

First line. Metadata only; not part of the entry tree.

```json
{"type":"session","version":3,"id":"uuid","timestamp":"2026-01-01T00:00:00.000Z","cwd":"/path/to/project"}
```

Forked sessions may include `parentSession`.

## Entry types

### `message`

Stores a user, assistant, tool-result, bash-execution, custom, branch-summary, or compaction-summary message.

```json
{"type":"message","id":"a1b2c3d4","parentId":null,"timestamp":"2026-01-01T00:00:01.000Z","message":{"role":"user","content":"Hello"}}
```

### `model_change`

Recorded when the active provider/model changes.

```json
{"type":"model_change","id":"d4e5f6g7","parentId":"c3d4e5f6","timestamp":"2026-01-01T00:05:00.000Z","provider":"openai","modelId":"gpt-5"}
```

### `thinking_level_change`

Recorded when the thinking/reasoning level changes.

```json
{"type":"thinking_level_change","id":"e5f6g7h8","parentId":"d4e5f6g7","timestamp":"2026-01-01T00:06:00.000Z","thinkingLevel":"high"}
```

### `compaction`

Stores a summary of older context plus the first entry kept verbatim.

```json
{"type":"compaction","id":"f6g7h8i9","parentId":"e5f6g7h8","timestamp":"2026-01-01T00:10:00.000Z","summary":"User discussed X...","firstKeptEntryId":"c3d4e5f6","tokensBefore":50000}
```

Optional fields:

- `details`: implementation-specific metadata, usually file tracking
- `fromHook`: `true` if generated by an extension

### `branch_summary`

Stores context from a branch abandoned during `/tree` navigation.

```json
{"type":"branch_summary","id":"g7h8i9j0","parentId":"a1b2c3d4","timestamp":"2026-01-01T00:15:00.000Z","fromId":"f6g7h8i9","summary":"Branch explored approach A..."}
```

### `custom`

Extension state. Not sent to the model.

```json
{"type":"custom","id":"h8i9j0k1","parentId":"g7h8i9j0","timestamp":"2026-01-01T00:20:00.000Z","customType":"my-extension","data":{"count":42}}
```

### `custom_message`

Extension-provided context. Sent to the model when on the active branch.

```json
{"type":"custom_message","id":"i9j0k1l2","parentId":"h8i9j0k1","timestamp":"2026-01-01T00:25:00.000Z","customType":"my-extension","content":"Injected context...","display":true}
```

### `label`

User-defined bookmark on an entry.

```json
{"type":"label","id":"j0k1l2m3","parentId":"i9j0k1l2","timestamp":"2026-01-01T00:30:00.000Z","targetId":"a1b2c3d4","label":"checkpoint-1"}
```

### `session_info`

Session metadata such as display name.

```json
{"type":"session_info","id":"k1l2m3n4","parentId":"j0k1l2m3","timestamp":"2026-01-01T00:35:00.000Z","name":"Refactor auth module"}
```

## Tree structure

- First entry has `parentId: null`.
- Each later entry points to its parent.
- Branching creates another child from an earlier entry.
- Active context is built by walking from current leaf to root.

```text
[user] ─ [assistant] ─ [user] ─ [assistant] ─┬─ [user] ← current leaf
                                              └─ [branch_summary] ─ [user]
```

## Context building

`buildSessionContext()`:

1. Walks from current leaf to root.
2. Applies latest model and thinking-level entries.
3. Inserts compaction summaries before kept messages.
4. Converts branch summaries and custom messages to model context.

## Parsing example

```typescript
import { readFileSync } from "node:fs";

const lines = readFileSync("session.jsonl", "utf8").trim().split("\n");

for (const line of lines) {
  const entry = JSON.parse(line);
  console.log(entry.type, entry.id ?? entry.version);
}
```

## `SessionManager` API

Common entry points:

- `SessionManager.create(cwd, sessionDir?)`
- `SessionManager.open(path, sessionDir?)`
- `SessionManager.continueRecent(cwd, sessionDir?)`
- `SessionManager.inMemory(cwd?)`
- `SessionManager.forkFrom(sourcePath, targetCwd, sessionDir?)`
- `SessionManager.list(cwd, sessionDir?, onProgress?)`
- `SessionManager.listAll(onProgress?)`

Common instance methods:

- `appendMessage(message)`
- `appendCompaction(summary, firstKeptEntryId, tokensBefore, details?, fromHook?)`
- `appendCustomEntry(customType, data?)`
- `appendCustomMessageEntry(customType, content, display, details?)`
- `appendLabelChange(targetId, label)`
- `branch(entryId)`
- `branchWithSummary(entryId, summary, details?, fromHook?)`
- `buildSessionContext()`
- `getTree()`
- `getSessionFile()`
